# FUQ Telegram Linux Control Bot

Status: `Beta`

Полностью рабочий Telegram-бот на `aiogram 3.x` для управления Linux-сервером через одно основное сообщение с `edit_text`.

## Что реализовано

- Главное меню: `СЕТЬ`, `ФАЙЕРВОЛ`, `СИСТЕМА`, `ФАЙЛЫ`, `БЭКАПЫ`, `ТЕРМИНАЛ`, `ИНСТРУМЕНТЫ`
- Режим единого окна через `InlineKeyboard` + `edit_text`
- FSM-сценарии для ввода IP, портов, PID, служб, путей
- Доступ для `ADMIN_ID` + дополнительных админов из `ADMIN_IDS` и через меню `Инструменты -> Админы`
- Пинг по IP и доменным именам
- Безопасное включение/выключение firewall с настраиваемыми безопасными портами
- Мониторинг и алерты с фоновым циклом
- Файервол-профили: `Web`, `SSH Only`, `DB Closed`, `Panic`
- Журналы: `kernel`, `auth`, `errors`, по службе, поиск
- Fail2ban: статус, jail-ы, ban/unban
- Docker/Compose: контейнеры, образы, compose-операции, логи
- Управление обновлениями пакетов
- Бэкапы PRO: создать, список, скачать, восстановить, удалить

## Структура

- `app/bot.py` — точка входа и сборка приложения
- `app/routers/` — разделенная логика по веткам меню
- `app/services/` — shell, metrics, storage, alerts, backups, updates
- `app/runtime.py` — управление единым окном и тасками метрик
- `install.sh` — полностью интерактивный автоустановщик

## Установка

```bash
chmod +x install.sh
sudo ./install.sh
```

Установщик спрашивает:

- `BOT_TOKEN`
- `ADMIN_ID`
- дополнительные `ADMIN_ID` (опционально)
- путь установки
- имя `systemd`-сервиса
- таймауты команд
- безопасные порты firewall
- включать ли firewall сразу
- ставить ли `Docker/Compose` и `Fail2ban`

После установки сервис поднимается автоматически.
